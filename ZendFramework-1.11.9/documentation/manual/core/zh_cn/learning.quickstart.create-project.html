创建你的项目

为了创建你的项目，你应该先下载并解压缩Zend Framework

安装 Zend Framework

获取Zend Framework最简单的方式是安装包含Zend Framework的PHP套装软件 » Zend Server。Zend Server 有原生安装包针对 Mac OSX, Windows, Fedora Core, 和 Ubuntu, 同样有通用安装包能兼容大多数Linux发行版.

在安装Zend Server后, 框架文件可能位于 /usr/local/zend/share/ZendFramework （Mac OSX 和 Linux版本）或者 C:\Program Files\Zend\ZendServer\share\ZendFramework （ Windows版本）. php的配置选项 include_path 已经配置成包含 Zend Framework了。

作为替代方案, 你可以 » 下载最新版Zend Framework 并解压缩；作好记录，记下你解压缩到什么位置了.

可选的操作是，你可以将框架下的子目录 library/ 添加到 php.ini中include_path的设定中。.

至此Zend Framework安装完成，并可以使用

创建你的项目

注意：zf 命令行工具
 在你的Zend Framework子目录 bin/ 中, 包含着 zf.sh 和 zf.bat 分别针对 Unix类(译者注：包含所有类Unix操作系统)用户 和 Windows用户，记下该脚本的绝对路径
如果你想无论什么路径下都能查看zf使用说明, 请替代绝对路径去访问脚本. 在类Unix系统上，你可以使用别名功能：alias zf.sh=path/to/ZendFramework/bin/zf.sh.
如果你在安装和使用zf工具方面有问题，请查阅 手册 .

打开终端 (Windows用户, Start -> Run, 输入cmd). 导航至你要创建项目的目录。然后使用恰当的脚本去执行下面的命令

  1. % zf create project quickstart

运行该命令将会创建基础的网站结构，包含初始化控制器和视图。目录结构如下：

  1. quickstart
  2. |-- application
  3. |   |-- Bootstrap.php
  4. |   |-- configs
  5. |   |   `-- application.ini
  6. |   |-- controllers
  7. |   |   |-- ErrorController.php
  8. |   |   `-- IndexController.php
  9. |   |-- models
  10. |   `-- views
  11. |       |-- helpers
  12. |       `-- scripts
  13. |           |-- error
  14. |           |   `-- error.phtml
  15. |           `-- index
  16. |               `-- index.phtml
  17. |-- library
  18. |-- public
  19. |   |-- .htaccess
  20. |   `-- index.php
  21. `-- tests
  22.     |-- application
  23.     |   `-- bootstrap.php
  24.     |-- library
  25.     |   `-- bootstrap.php
  26.     `-- phpunit.xml

此时，如果你没有将Zend Framework添加进入include_path，我们推荐你或者拷贝或者符号链接到 library/ 。 在某些情况下，你可能希望将Zend Framework中的library/Zend/递归拷贝或者符号链接到你项目目录下的library/中。 在类Unix操作系统上， 你将输入以下命令

  1. # Symlink:
  2. % cd library; ln -s path/to/ZendFramework/library/Zend .
  3.  
  4. # Copy:
  5. % cd library; cp -r path/to/ZendFramework/library/Zend .

Windows环境中使用图形化的Explorer将更加容易.

现在项目已经创建完成, 接下来的工作是理解：引导(bootstrap)、 配置(configuration)、动作控制器( action controllers)和视图

引导

你的引导类定义了什么样的资源和组件将被初始化。默认的，只有Zend Framework的 前端控制器 是被初始化的，它使用application/controllers/ 目录作为查找动作控制器的默认目录。 该类看起来像是这样的

  1. // application/Bootstrap.php
  2.  
  3. class Bootstrap extends Zend_Application_Bootstrap_Bootstrap
  4. {
  5. }

如你所见，很简洁，内容不多

配置

当Zend Framework很少被配置时，你总是需要配置自己的应用。默认配置位于以下位置：application/configs/application.ini，其中包含一些设置你PHP环境的基础指令（比如打开/关闭错误显示等），指示引导文件名字和位置， 和动作控制器的路径。下面是一个示例：

  1. ; application/configs/application.ini
  2.  
  3. [production]
  4. phpSettings.display_startup_errors = 0
  5. phpSettings.display_errors = 0
  6. includePaths.library = APPLICATION_PATH "/../library"
  7. bootstrap.path = APPLICATION_PATH "/Bootstrap.php"
  8. bootstrap.class = "Bootstrap"
  9. appnamespace = "Application"
  10. resources.frontController.controllerDirectory = APPLICATION_PATH "/controllers"
  11. resources.frontController.params.displayExceptions = 0
  12.  
  13. [staging : production]
  14.  
  15. [testing : production]
  16. phpSettings.display_startup_errors = 1
  17. phpSettings.display_errors = 1
  18.  
  19. [development : production]
  20. phpSettings.display_startup_errors = 1
  21. phpSettings.display_errors = 1

该文件中的某些部分你需要注意。 首先，当使用INI风格的配置时，你可以直接使用常量； APPLICATION_PATH是一个常用常量。对于以下部分你可能要注意： production,、staging, testing和 development.。这三个字段都将从 "production"继承变量。 这对于保持开发过程中的设置的组织性和一致性很有好处。

动作控制器

你的应用中的动作控制器将保存你的应用的工作流，和从请求到模型和视图的映射关系

一个动作控制器至少有一个以Action结尾的方法；这些方法可以通过web访问的方法调用。 默认情况下， Zend Framework URLs 允许/controller/action这种规则，其中的"controller" 映射至动作控制器的名字 (除去"Controller"的后缀)并且"action"映射至动作方法 (除去"Action" 后缀).

典型的，你总是需要一个IndexController， 这是一个可靠的控制器并且总是为网站的首页服务，还有一个 ErrorController，它总是指示诸如HTTP 404错误(比如控制器或动作没有找到等)和 HTTP 500 错误 (应用程序错误) 这样的事情

下面是一个默认的IndexController：

  1. // application/controllers/IndexController.php
  2.  
  3. class IndexController extends Zend_Controller_Action
  4. {
  5.  
  6.     public function init()
  7.     {
  8.         /* Initialize action controller here */
  9.     }
  10.  
  11.     public function indexAction()
  12.     {
  13.         // action body
  14.     }
  15. }

下面是默认的ErrorController：

  1. // application/controllers/ErrorController.php
  2.  
  3. class ErrorController extends Zend_Controller_Action
  4. {
  5.  
  6.     public function errorAction()
  7.     {
  8.         $errors = $this->_getParam('error_handler');
  9.  
  10.         switch ($errors->type) {
  11.             case Zend_Controller_Plugin_ErrorHandler::EXCEPTION_NO_ROUTE:
  12.             case Zend_Controller_Plugin_ErrorHandler::EXCEPTION_NO_CONTROLLER:
  13.             case Zend_Controller_Plugin_ErrorHandler::EXCEPTION_NO_ACTION:
  14.  
  15.                 // 404 error -- controller or action not found
  16.                 $this->getResponse()->setHttpResponseCode(404);
  17.                 $this->view->message = 'Page not found';
  18.                 break;
  19.             default:
  20.                 // application error
  21.                 $this->getResponse()->setHttpResponseCode(500);
  22.                 $this->view->message = 'Application error';
  23.                 break;
  24.         }
  25.  
  26.         $this->view->exception = $errors->exception;
  27.         $this->view->request   = $errors->request;
  28.     }
  29. }

你应该注意到(1)中的IndexController并未包含真实代码， (2) 中的ErrorController引用了"view"部分。这很好的引导我们进入下一个专题

视图

Zend Framework中的视图是使用平常的PHP老式语法写成的。视图脚本位于 application/views/scripts/， 它们已经按照控制器名字为目录的命名方式分类好了。 在我们的例子中，我们拥有IndexController和ErrorController，所以我们就有了相应的视图脚本的子目录index/ 和 error/ 。在这些子目录中，你可以创建相应的和控制器动作关联的视图脚本；默认情况下，我们拥有index/index.phtml和error/error.phtml两个视图脚本。

视图脚本包含任何你想写入的标记，想要插入PHP指令请使用标准的分割符号<?php和?>。

创建一个虚拟主机

为了使用快速开始我们推荐使用 » Apache web server。 Zend Framework 能够完美的工作在很多其他网络服务器上，包括Microsoft Internet Information Server、lighttpd,、nginx等等，但绝大多数开发人员至少对Apache非常熟悉，它能对Zend Framework的目录结构做出很好的介绍和良好的规则重写能力

为了创建你的虚拟主机你应该能定位httpd.conf 文件的位置，并找到其他配置文件的位置。一些通用的位置：

  • /etc/httpd/httpd.conf (Fedora, RHEL, and others)

  • /etc/apache2/httpd.conf (Debian, Ubuntu, and others)

  • /usr/local/zend/etc/httpd.conf (Zend Server on *nix machines)

  • C:\Program Files\Zend\Apache2\conf (Zend Server on Windows machines)

在httpd.conf (或者是某些系统中的httpd-vhosts.conf )中你需要做两件事情。 第一，确定NameVirtualHost已经定义；通常情况下， 你会定义成："*:80"。第二步，定义一个虚拟主机：

  1. <VirtualHost *:80>
  2.     ServerName quickstart.local
  3.     DocumentRoot /path/to/quickstart/public
  4.  
  5.     SetEnv APPLICATION_ENV "development"
  6.  
  7.     <Directory /path/to/quickstart/public>
  8.         DirectoryIndex index.php
  9.         AllowOverride All
  10.         Order allow,deny
  11.         Allow from all
  12.     </Directory>
  13. </VirtualHost>

有几件事情，你需要注意。首先，注意将DocumentRoot设置成我们项目中的子目录public；那表示只有那个目录下的文件才能被web访问。 其次注意AllowOverride, Order和 Allow指令；这些指令允许我们在项目中使用htacess文件。 开发中，这是不错的实践，给我们提供了无需重启服务器就能改变服务器指令的方法；然而在生产服务器中，我们要把htaccess文件的内容写入服务器配置，并且禁止htaccess。 第三需要注意SetEnv指令.。我们这是在设置虚拟主机的环境变量； 该变量将会被index.php捕获并且将用于设置Zend Framework 应用中的APPLICATION_ENV 常量的内容。生产环境中，你可以忽略这个指令 (一旦忽略，其默认值为"production") 或者直接设置成 "production".

最后你需要在你的hosts文件中添加一个和ServerName指令相关的项。Unix类操作系统中，它位于/etc/hosts；Windows中它通常位于C:\WINDOWS\system32\drivers\etc。下面的hosts的内容和系统类型无关

  1. 127.0.0.1 quickstart.local

当你准备好了，你要启动或者重启服务器
